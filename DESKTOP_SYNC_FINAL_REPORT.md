## 🔧 技术架构

### 通信协议
- **WebSocket** - 端口53318，用于数据传输
- **UDP多播** - 端口53317 (239.255.42.99)，用于设备发现
- **消息格式** - JSON + 换行符分隔

### 安全机制
- **配对码验证** - 6位数字，5分钟自动过期
- **哈希校验** - SHA256验证文件完整性
- **局域网限制** - 仅监听本地网络

### 同步算法
```
对每个todo ID：
  1. 仅本地有 → 发送到移动端
  2. 仅移动端有 → 从移动端接收
  3. 双方都有：
     - 哈希相同 → 跳过
     - 哈希不同 → 比较updatedAt
       - 本地更新 → 发送
       - 移动端更新 → 接收
       - 时间戳相同但哈希不同 → 冲突（记录警告）
```

## 🎨 UI特性

### 状态机（5个状态）
1. **waiting** - 等待移动端连接
2. **pairing** - 显示6位配对码（大字号 + 倒计时）
3. **syncing** - 显示进度条和文件计数
4. **complete** - 显示统计信息，2秒后自动关闭
5. **error** - 显示错误信息，提供重试按钮

### 工具栏集成
```
[搜索] [排序] [视图] [日历] [复盘] [🔄同步] [新建] [设置]
```

### 视觉设计
- **图标** - WifiOutlined (等待), SyncOutlined (同步), CheckCircle (成功), CloseCircle (失败)
- **配对码** - 48px粗体，字间距8px，蓝色高亮
- **进度条** - 动态百分比，实时更新
- **统计信息** - 发送/接收/跳过文件数

## 🚀 如何使用

### 1. 启动应用
```bash
npm run dev
```

### 2. 打开同步功能
- 点击工具栏的"同步"按钮（🔄图标）
- 等待SyncModal打开并显示"等待移动端连接..."

### 3. 移动端连接
- 在移动端打开同步功能
- 移动端应能通过UDP发现桌面设备
- 点击连接，桌面端显示6位配对码

### 4. 配对和同步
- 在移动端输入配对码
- 等待同步完成
- 查看同步统计信息
