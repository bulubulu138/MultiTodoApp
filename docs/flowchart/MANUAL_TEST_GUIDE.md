# 流程图功能手动测试指南

## 测试环境准备
1. 启动应用: `npm start`
2. 打开开发者工具查看控制台

## Task 12: 分享功能测试

### 12.1 生成分享链接
1. 创建一个包含 3-5 个节点的流程图
2. 点击工具栏的"分享"按钮
3. 选择"生成分享链接"
4. **预期结果**: 
   - 显示"分享链接已复制到剪贴板"消息
   - 链接已在剪贴板中
   - 如果流程图较大，显示警告信息

### 12.2 URL 长度检查
1. 创建一个包含 20+ 节点的大型流程图
2. 点击"分享" → "生成分享链接"
3. **预期结果**: 显示"链接较长"的警告

### 12.3 导出图片分享
1. 点击"分享" → "导出为图片"
2. **预期结果**: PNG 图片已下载

## Task 13: 性能优化和错误处理测试

### 13.1 防抖保存
1. 快速拖动多个节点
2. 观察控制台
3. **预期结果**: 
   - 保存操作延迟 500ms 执行
   - 不会频繁触发保存

### 13.2 大规模流程图警告
1. 创建超过 100 个节点的流程图
2. 观察控制台
3. **预期结果**: 显示性能警告

### 13.3 错误处理
1. 尝试导出空流程图
2. 尝试在没有画布时导出图片
3. **预期结果**: 显示友好的错误提示

### 13.4 ErrorBoundary
1. 故意触发错误（如删除必需的节点数据）
2. **预期结果**: 显示错误页面，不会崩溃

## 性能指标验证
- 打开控制台
- 执行: `PerformanceMonitor.getReport()`
- 检查各项操作的平均耗时
