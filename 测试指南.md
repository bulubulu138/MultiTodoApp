## 常见问题排查

### 问题1: 仍然只显示1个待办

**排查步骤：**
1. 检查主进程日志，查找 `FileStorageManager.getAllTodos` 的日志
2. 查找是否有 `Failed to load todo` 或 `UUID not found in map` 的错误
3. 运行 `checkDataIntegrity()` 检查数据一致性
4. 如果不一致，运行 `repairUuidMapping()` 修复

---

### 问题2: UUID映射文件损坏

**症状：**
- 日志显示 `Error reading UUID map`
- `checkDataIntegrity()` 返回 `isValid: false`

**解决方案：**
```javascript
// 1. 备份现有映射（如果有）
// 备份会自动创建在 uuid-to-file.json.backup

// 2. 运行修复
await window.electronAPI.debug.repairUuidMapping();

// 3. 重建索引
await window.electronAPI.debug.rebuildIndex();

// 4. 重新加载应用
location.reload();
```

---

### 问题3: 索引构建失败

**症状：**
- 日志显示多个 `Failed to normalize` 错误
- `buildIndex` 报告大量失败

**解决方案：**
```javascript
// 1. 检查具体的错误日志
// 在主进程Console中查看详细错误信息

// 2. 强制重建索引
await window.electronAPI.debug.rebuildIndex();

// 3. 如果仍然失败，检查markdown文件格式是否正确
// 文件应该包含有效的YAML frontmatter和id字段
```

---

### 问题4: 性能问题（加载缓慢）

**症状：**
- 应用启动时间很长
- 待办加载超过5秒

**解决方案：**
1. 检查是否有大量待办（>100个）
2. 查看日志中的 `[Performance]` 时间统计
3. 如果超过10秒，可能需要优化索引

## 回滚方案

如果修复导致问题，可以快速回滚：

### 方案1: 切换回数据库模式

1. 打开应用设置
2. 在存储设置中选择"数据库模式"
3. 重启应用

### 方案2: 手动恢复映射

```bash
# 1. 备份当前映射
cp .multitodo-metadata/uuid-to-file.json .multitodo-metadata/uuid-to-file.json.broken

# 2. 恢复备份（如果存在）
cp .multitodo-metadata/uuid-to-file.json.backup .multitodo-metadata/uuid-to-file.json

# 3. 重启应用
```

### 方案3: 重新迁移

```javascript
// 1. 切换回数据库模式（在应用设置中）

// 2. 重新执行迁移到Markdown存储
// 在应用设置中选择"文件存储模式"并指定路径

// 3. 重启应用
```

## 开发者调试技巧

### 1. 查看完整的待办加载过程

```javascript
// 在Console中执行，观察详细的加载过程
const originalLog = console.log;
console.log = function(...args) {
  if (args[0] && args[0].includes && args[0].includes('FileStorageManager')) {
    originalLog.apply(console, args);
  }
};

// 然后重新加载待办
await window.electronAPI.todo.getAll();

// 恢复正常日志
console.log = originalLog;
```

### 2. 监控文件系统变化

```javascript
// 查看文件监听器状态

```

### 3. 导出诊断报告

```javascript

  diagnostic,
  integrity,
  todoCount: todos.length,
  timestamp: new Date().toISOString()
```
